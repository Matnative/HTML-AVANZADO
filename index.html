Etiquetas semánticas estructurales.

Inicio.
El DIV antes se usaban las div's para separar diferentes secciones, ya que engloba un contenido de bloque genérico, pero aunque esté aquí no representa ningun poder a nivel semántico.

El HEADER actúa como cabecera de nuestro sitio web.

NAV hace referencia a una navegación, se suele usar para para agrupar links.

MAIN va a definir la sección principal del documento, sólo puede existir una por documento.

El FOOTER haría referencia al pié de página de un sitio web o de una sección.

ARTICLE nos sirve para representar una sección de autocontenido, significa que por sí sola explica todo el contenido independientemente a la interface del sitio.

ASIDE va a dirigirse a un contenido secundario o complementario, típico "Te puede interesar..." o diferentes secciones relacionadas o no al contenido principal.

La SECTION representa una sección de contenido genérico

ADDRESS representa una dirección física o dirección de contacto.

Etiquetas de bloque y etiquetas de línea.

Inicio.
La etiqueta de bloque por excelencia es la DIV. Una etiqueta de bloque ocupa todo el espacio de la línea y es independiente a cualquier otro bloque.

Y la etiqueta de línea por excelencia es la SPAN. Una etiqueta de línea solo ocupa el espacio necesario que abarca su contenido. Otro span

3 maneras de aplicar hojas de estilo en cascada (CSS3/CASCADE STYLE SHEET).

Inicio.
  • Crear una etiqueta LINK en el head enlazado a un archivo CSS3.
  • Crear en el head una etiqueta STYLE, se podrá usar como hoja de estilo en cascada.
